Overview

genmdt is a Stata command designed to generate standardized multidimensional statistical tables from complex survey data. The command leverages Stata survey estimation procedures to compute indicators while properly accounting for survey weights, stratification, and clustering.

The generated output follows a machine-readable long-format structure suitable for:

  • Statistical databases
  • Open data portals
  • Dashboards
  • Automated reporting systems
  • Publication-ready tables

The command supports estimation using:

  • mean
  • total
  • ratio
  • median

Syntax

genmdt [varlist] [, 
    svy_options
    dimension_options
    formatting_options
] : command

Requirements

Before running genmdt, survey settings must be declared using svyset. 

svyset psuid [pweight=finalwgt], strata(stratid)
Using if or in restrictions directly may produce incorrect variance estimates for subpopulations. Use subpop() instead.

Disaggregation Dimensions

Variables specified after genmdt define the estimation domains or disaggregation dimensions. 

genmdt region sex rural, mean(age)

If no dimension variable is specified, indicators are estimated at the overall population level.

Margin Labels

The marginlabels() option allows simultaneous estimation of:

  • Disaggregated categories
  • Overall population margins
genmdt sex, ///
    marginlabels("sex@Both sexes") ///
    mean(age)

Estimation Parameters

Mean Estimation

genmdt region, mean(age income)

Total Estimation

genmdt region, total(population livestock)

Ratio Estimation

genmdt region, ratio(expenditure/income)
genmdt region, ///
    ratio(PovertyRate:poor/population)

Median Estimation

genmdt region, median(income)

Median estimation is internally performed using collapse.

Subpopulation Estimation

Use subpop() to estimate indicators for subpopulations while preserving correct variance estimation. 

genmdt region, ///
    subpop(female==1) ///
    mean(age)

Hierarchical Geographic Variables

Hierarchical geographic dimensions should be specified using hiergeovars(). 

genmdt , ///
    hiergeovars(region district village) ///
    mean(yield)

Optional geographic margin labels can be specified using: 

geomarginlabels("National")

Reporting and Formatting Options

Units

units("income@USD" "yield@kg/ha")

Indicator Labels

indicatornames(
    "income@Average household income"
    "yield@Average crop yield"
)

Integer Formatting

integer(population livestock)

Output Structure

The output generated by genmdt contains:

Variable Description
Dimensions Disaggregation variables
Indicator Estimated variable
Parameter Mean, total, ratio, median
Estimate Estimated value
SE Standard error
CV Coefficient of variation
CI Lower Lower confidence interval
CI Upper Upper confidence interval
N Unweighted sample size
Weighted N Weighted sample size

Examples

Example 1 — Mean Estimation

webuse nhanes2f

svyset psuid [pweight=finalwgt], strata(stratid)

genmdt region, mean(age)

Example 2 — Labels and Units

webuse nhanes2f

svyset psuid [pweight=finalwgt], strata(stratid)

genmdt region, ///
    marginlabels("region@USA") ///
    mean(age) ///
    units("age@Years") ///
    indicatornames("age@Average age of individuals")

Example 3 — Multiple Indicators

webuse nhanes2f

svyset psuid [pweight=finalwgt], strata(stratid)

genmdt region sex rural, ///
    marginlabels("region@USA" "sex@Both") ///
    mean(highbp diabetes) ///
    total(heartatk) ///
    integer(heartatk) ///
    units( ///
        "highbp@%" ///
        "diabetes@%" ///
        "heartatk@people" ///
    ) ///
    indicatornames( ///
        "highbp@Proportion with high blood pressure" ///
        "diabetes@Proportion with diabetes" ///
        "heartatk@Total heart attack cases" ///
    )

Best Practices

Always define survey settings using svyset before estimation.
Prefer subpop() over direct if restrictions.
Use indicator labels and units to improve readability and dissemination workflows.
Use hiergeovars() for hierarchical geographic structures.
